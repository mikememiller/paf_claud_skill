# PAF Agent Factory — Deliverables (technical, generic)

Every Private-Agent-Factory engagement produces the **same consistent,
Syntax-branded deliverable set**, regardless of domain. This document describes
*all* deliverables — what each is, who it's for, its format, its contents, and
how it's generated. It is **domain-neutral**; Oracle EBS is the bundled worked
example (the AP agent), but the catalog is identical for any agent.

**Two principles.**
1. **Single source of truth → no drift.** Customer-facing files are *generated*
   from the in-repo markdown (`docs/*.md`, `README.md`, `INSTALL.md`) + the one
   pricing model (`references/pricing-and-bom.md` → `pricing.py`). Edit the
   source, regenerate — the deck, SOW, and ROI never disagree.
2. **QA-gated.** Deliverables are generated **only after the QA gate is green**
   (`references/qa-and-bugfixing.md`) and visually re-checked before ship.

## A. Customer-facing deliverables

All generators live in `templates/deliverables/` and are **spec-driven**: they read
`spec.yaml` (via `output/spec.json` + `output/pricing.json`, produced by
`pricing.py`) + `assets/brand.json`. See **`samples/README.md`** for the run order
and **`samples/`** for ready examples.

### Detail
1. **Sales deck** — the *why*: business problem, the agentic solution, the
   architecture (hero diagram), ROI headline, deployment path. Hero graphics per
   `references/graphics-standard.md`.
2. **Installation guide** — prereqs, connection, deploy, register the MCP, import
   the `.paf`, rebind, smoke-test. Mirrors `INSTALL.md` + `setup.sh`.
3. **Technical design** — architecture, data flow, the deterministic policy, the
   interface contract, security posture, the QA strategy. Embeds **both** the
   architecture and functional-flow diagrams.
4. **SOW (comprehensive, stands alone)** — business problem, solution, value
   proposition, **functional-flow + technical-architecture** graphics, scope &
   deliverables, **ongoing governance**, **implementation timeline**, full **RACI**
   matrix, **SLAs**, **entitlements**, pricing (identical to deck/ROI), the **OCI
   BOM**, acceptance & assumptions, and a **signature block**. Generic legal shell
   — counsel reviews before use.
5. **OCI BOM** — sized line items (ADB/PAF, managed MCP, network, GenAI tokens)
   + a consumption estimator the buyer can flex.
6. **ROI calculator** — interactive payback/NPV from the same pricing inputs;
   a "use my data" mode for live numbers.
7. **PAF integration package** — the **importable `.paf`** (default password
   `simple4u`; `flow` or tool-bound `flowGraph`), the MCP tool/SQL-Report
   definitions, and the canvas recipe. See `core/paf-import.md`.

## C. How they're produced
- **Brand** — one source: `assets/brand.json` (palette, fonts, logo, tri-color
  motif) drives every generator (`references/branding.md`).
- **Pricing** — one model: `pricing.py` output feeds SOW, deck, BOM, and ROI so
  the numbers are identical everywhere.
- **Graphics** — hero diagrams per `references/graphics-standard.md` (SVG → PNG
  via `sharp`), embedded in the deck/design/SOW.
- **Tooling** — `.pptx` via pptxgenjs (hero pipeline) or `anthropic-skills:pptx`;
  `.docx`/`.xlsx` via the `anthropic-skills` `docx`/`xlsx` skills.

## D. Where deliverables are created
- **Catalog / spec / generators (reusable):** this skill — `references/deliverables.md`
  (recipe), `templates/deliverables/` (generators), `core/` (the `.paf` + gate).
- **Per-engagement artifacts (the actual files):** the agent's own git repo, in
  `deliverables/` (the `.docx`/`.xlsx`/`.html`/`.pptx`) and `slides/` (deck +
  hero-diagram generators). Example: the EBS AP agent's
  `EBS-Contract-Renewal-PAF/deliverables/` + `slides/`.

## E. The showcase deck
`deliverables/PAF_Factory_Deliverables.pptx` (built by `build_deck.js` +
`build_diagram.js`, pptxgenjs hero pipeline) is the **wow-graphics deck that
describes this entire catalog** — use it to walk a customer or exec through what
every engagement ships. Regenerate: `node deliverables/build_diagram.js &&
node deliverables/build_deck.js` (needs `pptxgenjs` + `sharp`).
